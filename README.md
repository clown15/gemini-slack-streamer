# Gemini Slack Streamer

이 프로젝트는 Google Gemini CLI를 Slack과 연동하여 강력한 AI 에이전트 기능을 채팅창에서 사용할 수 있게 해주는 도구입니다. `tmux`와 `PTY`를 활용하여 에이전트의 상태를 지속적으로 유지하고, 고신뢰성 동기화 엔진을 통해 실시간 터미널 출력을 슬랙 스레드로 안전하게 스트리밍합니다.

## 주요 기능

- **지속적 세션 관리**: `tmux`를 사용하여 봇 서버가 재시작되어도 에이전트의 작업 문맥(Context)을 유지합니다.
- **고신뢰성 스트리밍 (Robust Sync)**:
    - **원자적 체이닝(Atomic Chaining)**: 답변이 슬랙 메시지 제한(3,000자)을 초과할 경우, 데이터 유실이나 덮어쓰기 없이 새로운 메시지를 자동으로 생성하여 연결합니다.
    - **경계 격리(Command Isolation)**: 매 명령어마다 고유한 유니크 마커(Unique Marker)를 주입하여, 터미널 히스토리가 남아있어도 이전 답변이 섞이지 않는 깨끗한 결과를 보장합니다.
    - **유니코드 무결성**: 모든 텍스트 처리를 `rune` 단위로 수행하여 한글 등 멀티바이트 문자의 깨짐 현상을 완벽히 차단합니다.
- **이모지 상태 표시**: 작업 시작 시 ⏳(모래시계), 완료 시 ✅(확인 표시) 이모지 반응을 통해 진행 상태를 직관적으로 알 수 있습니다.
- **자동 초기화 가드**: MCP(Model Context Protocol) 서버 로딩 등 초기화 과정을 자동으로 감지하고 준비될 때까지 입력을 대기시킵니다.
- **터미널 노이즈 필터링**: 스피너, 작업 시간 안내, 시스템 상태바 등 터미널 고유의 노이즈를 본문에서 깔끔하게 제거합니다.

## 설치 및 실행

### 요구 사항
- Docker 및 Docker Compose
- Slack App (Socket Mode 활성화)
- `tmux` (컨테이너 내부에 자동 설치됨)

### 환경 설정 (`.env`)
`.env.example` 파일을 복사하여 `.env` 파일을 생성하고 필요한 토큰을 입력합니다.

```env
SLACK_BOT_TOKEN=xoxb-...
SLACK_APP_TOKEN=xapp-...
ALLOWED_CHANNEL_ID=C...
GEMINI_BIN_PATH=gemini
```

### 실행
```bash
docker compose up -d --build
```

## 사용 방법

슬랙의 지정된 채널에서 `!` 접두사를 사용하여 명령어를 입력합니다.

- `!/sg:help`: 도움말 확인
- `!/sg:design [시스템명]`: 시스템 설계 요청
- `!/sg:implement [기능]`: 코드 구현 요청
- `!/sg:troubleshoot [이슈]`: 버그 및 문제 진단
- `!reset` 또는 `!초기화`: 현재 세션 및 터미널 히스토리를 완전히 초기화

## 아키텍처

1. **Slack Events**: Socket Mode를 통해 사용자의 명령어를 수신합니다.
2. **Tmux Manager**: 각 채널별로 독립적인 `tmux` 세션을 관리하며, 명령어별로 유니크 마커를 통해 답변 구역을 격리합니다.
3. **PTY Bridge**: 에이전트의 터미널(PTY) 출력을 캡처하여 Go 백엔드로 전달합니다.
4. **Atomic Adaptive Sync**: 터미널 화면의 변화를 실시간 감지하며, 메시지 분할 전송 시 상태 불일치가 발생하지 않도록 원자적(Atomic)으로 메시지 상태를 전환합니다.
5. **Emoji Feedback**: 원본 메시지에 이모지 반응을 추가/제거하여 작업 상태를 표시합니다.
